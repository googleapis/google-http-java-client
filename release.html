Release Process

First time set up

  • Make sure you have at least a Committer role
  • Make sure you have permission to post to the Announcement Blog.
  • Download JDiff.
  • Set up for pushing to central Maven repository
    • Sign up for a Sonatype JIRA account at https://issues.sonatype.org. Click Sign Up in the Login box, fill out the simple form, and click Sign Up.
    • Install a GPG client from http://www.gnupg.org/, and then create a GPG key using these instructions, making sure to specify a passphrase to be used every time you release. Make sure to run the instructions under "Distribute Your Public Key".
    • Add these lines to your ~/.hgrc: 
      
[ui]
username = Your Name <yourname@domain.com>

[auth]
google-http-java-client.prefix = https://google-http-java-client.googlecode.com/hg/
google-http-java-client.username = [username from https://code.google.com/hosting/settings]
google-http-java-client.password = [password from https://code.google.com/hosting/settings]
javadoc-google-http-java-client.prefix = https://javadoc.google-http-java-client.googlecode.com/hg/
javadoc-google-http-java-client.username = [username from https://code.google.com/hosting/settings]
javadoc-google-http-java-client.password = [password from https://code.google.com/hosting/settings]
wiki-google-http-java-client.prefix = https://wiki.google-http-java-client.googlecode.com/hg/
wiki-google-http-java-client.username = [username from https://code.google.com/hosting/settings]
wiki-google-http-java-client.password = [password from https://code.google.com/hosting/settings]
    • Add these lines to your ~/.m2/settings.xml (create this file if it does not already exist): 
      <settings>
  <servers>
    <server>
      <id>google-snapshots</id>
      <username>[username for https://issues.sonatype.org]</username>
      <password>[password for https://issues.sonatype.org]</password>
    </server>
    <server>
      <id>google-releases</id>
      <username>[username for https://issues.sonatype.org]</username>
      <password>[password for https://issues.sonatype.org]</password>
    </server>
    <server>
      <id>sonatype-nexus-snapshots</id>
      <username>[username for https://issues.sonatype.org]</username>
      <password>[password for https://issues.sonatype.org]</password>
    </server>
    <server>
      <id>sonatype-nexus-staging</id>
      <username>[username for https://issues.sonatype.org]</username>
      <password>[password for https://issues.sonatype.org]</password>
    </server>
  </servers>
</settings>

Fork version 1.7.1-beta

This is done in preparation for release of the 1.7 branch 1-2 weeks before the official launch. We create a named Mercurial branch for 1.7. Development in the default Mercurial branch may then progress for the next minor version.
  • Create the Mercurial named branch: 
    hg branch 1.7
hg commit -m "create 1.7 branch in preparation for release"
hg push --new-branch
hg update default
  • For the following files, replace 1.7.1-beta with the next minor version: (TODO: update this section to use sed)
    pom.xml
*/pom.xml
jdiff.xml (first change "1.7" to next minor version, then change "1.6" to "1.7")
release.html (first change "1.7" to next minor version, then change "1.6" to "1.7")
google-http-client/src/main/java/com/google/api/client/util/Strings.java
  • Commit and push: 
    hg commit -m "Start next minor version"
hg push

Release version 1.7.1-beta

This process is followed for the official launch. We are releasing the code from the 1.7 Mercurial branch.
  • Checkout the source, update dependencies.html, update the version constant, and push to central Maven repository (based on instructions from Sonatype OSS Maven Repository Usage Guide):
    cd /tmp
rm -rf google-http-java-client
mkdir google-http-java-client
cd google-http-java-client
hg clone -b 1.7 https://google-http-java-client.googlecode.com/hg/ 1.7
cd 1.7

mvn project-info-reports:dependencies
cp google-http-client/target/site/dependencies.html google-http-client-assembly/dependencies/dependencies.html
cp google-http-client-android2/target/site/dependencies.html google-http-client-assembly/dependencies/android2-dependencies.html
cp google-http-client-android3/target/site/dependencies.html google-http-client-assembly/dependencies/android3-dependencies.html
cp google-http-client-appengine/target/site/dependencies.html google-http-client-assembly/dependencies/appengine-dependencies.html

sed s/-SNAPSHOT//g <google-http-client/src/main/java/com/google/api/client/util/Strings.java >/tmp/Strings.java
cp /tmp/Strings.java google-http-client/src/main/java/com/google/api/client/util
hg commit -m "prepare 1.7.1-beta for release"
hg push

mvn clean deploy
mvn release:clean
mvn release:prepare -DautoVersionSubmodules=true -Dtag=1.7.1-beta --batch-mode
mvn release:perform -Darguments=-Dgpg.passphrase=[YOUR GPG PASSPHRASE]
  • Release on oss.sonatype.org
    • Log in to https://oss.sonatype.org
    • Click on "Staging Repositories" and look for the "com.google" Repository with your username
    • check box next to uploaded release; click Close; click Close
    • check box next to uploaded release; click Release; click Release
    Central Maven repository is synced hourly. Once uploaded onto repo2.maven.org, it will be found at: repo2.maven.org
  • Start development of next bug fix version, by replacing 1.7.1-beta with the next bug fix version:
    NEXT_VERSION=(next bug fix version 1.7.X-beta)
sed s/1.7.1-beta/$NEXT_VERSION/g <jdiff.xml >/tmp/jdiff.xml
sed s/1.7.1-beta/$NEXT_VERSION/g <release.html >/tmp/release.html
sed s/1.7.1-beta/$NEXT_VERSION-SNAPSHOT/g <google-http-client/src/main/java/com/google/api/client/util/Strings.java >/tmp/Strings.java
mv /tmp/jdiff.xml /tmp/release.html .
mv /tmp/Strings.java google-http-client/src/main/java/com/google/api/client/util
hg commit -m "start $NEXT_VERSION"
hg push
  • Prepare the released library zip file and javadoc
    • Overview: JDiff is a tool for showing the difference between the JavaDoc of one version of the library vs. the JavaDoc of an old version of the library. Hence, we will check out the last minor version of the library into a temporary directory and compare against the JavaDoc of the just released version of the library, and then hg commit that diff into the javadoc Hg repository.
    cd /tmp/google-http-java-client
hg clone https://code.google.com/p/google-http-java-client.javadoc/ javadoc
hg clone https://code.google.com/p/google-http-java-client.wiki/ wiki
hg clone -r 1.6.0-beta https://google-http-java-client.googlecode.com/hg/ 1.6.0-beta
hg clone -r 1.7.1-beta https://google-http-java-client.googlecode.com/hg/ 1.7.1-beta

cd 1.7.1-beta
mvn javadoc:jar install site
cp -R target/site/apidocs ../javadoc/1.7.1-beta
change these constants from jdiff.xml for your environment: JDIFF_HOME, MAVEN_REPOSITORY_HOME
ant -f jdiff.xml

cd ../javadoc
hg add
PREV_VERSION=(previous bug fix version 1.6.X-beta)
sed s/$PREV_VERSION/1.7.1-beta/g /tmp/index.html && mv /tmp/index.html index.html
hg commit -m "1.7.1-beta"
hg push

cd ../wiki
for s in *; do sed s/$PREV_VERSION/1.7.1-beta/g <$s >/tmp/$s && mv /tmp/$s .; done
hg revert --no-backup ReleaseNotes.wiki
edit ReleaseNotes.wiki to add release notes for 1.7.1-beta
hg commit -m "1.7.1-beta"
hg push
  • WAIT! Don't do the following instructions until after the library has been uploaded to repo2.maven.org at: repo2.maven.org
  • Update to new version on http://code.google.com/p/google-http-java-client
    • Upload to http://code.google.com/p/google-http-java-client/downloads/entry
      • Summary: Google HTTP Client Library for Java, version 1.7.1-beta
      • File: /tmp/google-http-java-client/1.7.1-beta/google-http-client-assembly/target/google-http-client-1.7.1-beta-java.zip
      • Labels: Type-Archive, OpSys-All, and Featured
      • click Submit file
    • If it is a bug fix release, deprecate the library from any previous versions by removing any Featured label and adding the Deprecated label.
  • New Post on the announcement blog about the new version, including links to new features and bugs fixed.
